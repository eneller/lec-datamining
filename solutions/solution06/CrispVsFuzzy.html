Crisp vs. Fuzzy $k$-means¶

Part 1: Computational Performance¶

It is to be expected that the fuzzy method requires more CPU time since we have a full dependency between every data point and every cluster prototype. For example, we need to calculate the membership values for every (point, prototype) combination. The attached paper has some benchmarks which show that this is indeed the case.

Part 2: Noise Influence¶

  • Crisp clustering reacts instable to the noise. It can happen that the cluster centres jump around (e.g. for 64-65-66 noise points) even though just one more point has been added to the scene. At around 85 noise points, we even get a completely different clustering result.
  • On the other hand, fuzzy clustering reacts more stable. The noise does have an influence but it is more smooth and we don't jump around so much anymore. What is more, we remain with the same cluster configuration until the end (no completely different clustering results).
  • The cluster centre of the crisp method moves ahead of the fuzzy method because each new point has full influence in the prototype position. In a way, crisp clustering is sensitive for outliers. This is less problematic for fuzzy clustering since distant points have only little influence.

Image Segmentation¶

In [4]:
def cluster_image(image, n_clusters, fuzzifier):
    # Cluster the image pixels (colours)
    data = image.reshape(-1, 3)
    centroids = skfuzzy.cmeans(data.transpose(), c=n_clusters, m=fuzzifier, error=0.0001, maxiter=2000, seed=1)[0]
    
    # Find the closest cluster for each pixel
    nbrs = NearestNeighbors(n_neighbors=1).fit(centroids)
    cluster_labels = nbrs.kneighbors(data)[1]

    # Replace each pixel with the corresponding centroid colour
    image_seg = centroids[cluster_labels]
    image_seg = image_seg.reshape(image.shape)
    
    return image_seg

Interpretation¶

  • With the crisp clustering method, the results change quite a lot. The background structure is not always detected well and the colour inside the flowers varies (much more saturated brown). The results also depend heavily on the used seed.
  • The fuzzy method, on the other hand, is relatively stable. The detected structure and the colours stays roughly the same. Also, the results vary less with when using different seeds.

It is to note that the noise pixels are outliers in the colour space (since salt & pepper produces extreme values):

Especially the crisp method is influenced by these outliers. However, even though the fuzzy method gets less distorted by these outliers, this has only an effect on the resulting colour values. It does not mean that the image pixels are now noise free. This is not possible with fuzzy $k$-means and also not intended. The noise colours still get assigned to one of the clusters resulting in a noise image.
